journalctl

  • Used to query the systemd journal written by systemd-journald.service.
  • Stored persistently at /var/log/journal (if enabled) or at /run/log/journal/ (lost during reboot). To change journal location, use Storage= in journald.conf
  • All outputs from a systemd service (to stdout and stderr) are stored to the journal automatically.
  • Fields prefixed with underscore("_") are trusted fields(implicitly added)
  • Multiple fields can be specified(logical AND, if "+" is not specified) separated by space or same field with different values(logical OR).
  • Refer man systemd.journal-fields for different fields and their descriptions.
  • e.g.To list logs related to user ID : 1000
    journalctl _UID=1000
  • The fields _COMM _EXE _CMDLINE to filter logs according to name, executable path, command line of process.
  • _TRANSPORT to filter based on how the entry was received. Possible values are : audit, driver, stdout, kernel, syslog, journal.
  • Use "-p" to list all logs of priority given and above. Refer manual for various priorities.
  • Use "-b" to filter logs based on boot. "0" is the current boot(if it's on the current machine). Or use _BOOT_ID
  • To output the logs in different formats like json, json-pretty use option "-o".
  • To remove archived file use --vacuum-size= {K, M, T} or use --vacuum-time= to remove logs based on time {s, m, h, days, weeks, months, years}.
